User Documentation

Getting up and running with your analytics enviroment

Introduction

SEMOSS is a data-driven application that enables users to explore relationships and connections in an interactive format through intermediate conduits, i.e. transitive relationships. This document provides general usage instructions and guidance in deployment of the open source components of SEMOSS, in contrast to the developer guide, which provides instructions and guidance in terms of development "on top of" the tool.

Overview

    This document contains:
  • An overview of the out-of-the-box SEMOSS user interface (Section 1).
  • Instructions for creating and loading a new database and developing the ontological structure (metamodel) for the database (Section 2).
  • Limited guidance in querying databases to understand advanced relationships using the SPARQL query language (Section 3).
  • A use case in deploying “out of the box” SEMOSS to analyze the sample security database (Section 4).

1: Getting Started

In order to begin working on SEMOSS, there are a number of files that must be downloaded and programs that must be installed.

This section lays out what each of the steps are that must be taken to begin editing the code.

1.1 Installation

SEMOSS has been built using the Eclipse integrated development environment and it is therefore advisable to use the latest version of Eclipse for editing the code.

Eclipse is available for download at http://www.eclipse.org/downloads/index-developer.php

A Java Development Kit (JDK) is also necessary to open SEMOSS for editing.

While most of the SEMOSS functionality works with a 64-bit JDK, it is recommended to use 32-bit to ensure all of the features work correctly.

The latest JDK is available for download at http://www.oracle.com/technetwork/java/javase/downloads/index.html

The source code of SEMOSS is available for download from GitHub which can be accessed through the site http://semoss.org/
