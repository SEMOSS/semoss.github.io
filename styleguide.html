Coding Guidelines

Introduction

The following rules are laid out in order to help our team maintain the CSS repository within the SEMOSS Web application. Before making a commit make sure your CSS follows these guidelines.

Table of Contents

  • Folder Structure
  • File Structure
  • Selector Structure
  • IDs
  • RECESS

Folder Structure

All css that you must edit or work with will go into the css/custom folder.

All css that is not inside the custom folder must remain unedited. These are libraries and not to be altered unless a newer version is available or they are no longer necessary.

File Structure

Visualizations that need CSS Styling should have a unique css file.

Partial specific CSS can go into style.css within the appropriate section. If the CSS is partial specific put it under the correct comment heading.

At the top of each file there should be a commented explanation of what the CSS file is responsible for.

Selector Structure

We are using a 2 space soft tab for our CSS properties. The property order is defined by RECESS but follows this general outline.

  1. Position
  2. Sizing
  3. Box Model
  4. Fonts
  5. Decorations

You can view the full RECESS order list here.

ID's

We are using ID's for javascript hooks and internal links, nothing more. There should never be an ID in our custom CSS stylesheets.

RECESS

We are using Twitter's RECESS for our CSS linting. In order to get up and running with it you need to follow these steps:

  1. Download the latest version of node
  2. Run npm install to install dependancies
  3. Run npm install -g grunt-cli to install grunt
  4. Run grunt rec to lint your CSS.

Make sure to run the grunt rec command before you commit CSS. RECESS will only check the custom folder. If your order is incorrect it will let you know the correct order.
You can copy this to your clipboard by right clicking on your command prompt, selecting Mark, highlighting the text, and then pressing enter.
Take a look at the website for a full list of rules. The only one we have disabled is noOverqualifying.
